## Project Overview

`phnx` is a Python meshing library for nuclear workflows. It wraps the commercial **Simmetrix SimModSuite** (SMS) library via C++ bindings using **nanobind**, and exports meshes using **Gmsh**. The Python package name is `phnx`; the C extension module is `_core`.

## Build Commands

The project uses **pixi** for environment management and **CMake** for building.

```bash
# Configure (run once, or after CMakeLists changes)
make conf
# Equivalent: cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug -DCMAKE_PREFIX_PATH="${CONDA_PREFIX}" -DCMAKE_EXPORT_COMPILE_COMMANDS=TRUE

# Build and install the Python extension into phnx/_core/
make build
# Equivalent: cmake --build build -j 8 -t install

# Clean build artifacts
make clean
# Removes: build/, phnx/_core/, _deps/
```

To activate the pixi environment before building or running:
```bash
pixi shell
```

## Architecture

### Build Pipeline

1. **nanobindgen** (external editable package at `../nanobindgen`) reads `src/phnx.h` and auto-generates `src/autogen/bind_phnx.h` ŌĆö the nanobind class/method registration code. The docstrings and binding annotations live in `src/phnx.h` as special `@nb` doc comments.
2. CMake builds a `main` static library from `src/phnx.cpp` + `src/init.cpp`.
3. CMake builds the `_core` nanobind module from `src/python/nbphnx.cpp`, which includes `src/autogen/bind_phnx.h` and links against `main`.
4. The module is installed to `phnx/_core/`, making it importable as `phnx._core._core`.

### Source Layout

- `src/phnx.h` / `src/phnx.cpp` ŌĆö Core C++ classes: `Model`, `MeshCase`, `Mesh`, `SurfaceMesh`, `VolumeMesh`, `ModelItem`, `Entity`, `Part`, `Assembly`, `Vertex`, `Edge`, `Face`, `Region`
- `src/init.h` / `src/init.cpp` ŌĆö SMS library initialization/termination, license loading, message handler routing to spdlog
- `src/python/nbphnx.cpp` ŌĆö Nanobind module entry point; calls `init()`, sets up intrusive ref-counting, registers `bind_phnx`
- `src/autogen/bind_phnx.h` ŌĆö **Auto-generated**; do not edit manually. Regenerated by `nanobindgen` from `src/phnx.h`
- `phnx/__init__.py` ŌĆö Python package entry; imports `Mesh`, `MeshCase`, `Model`, `SurfaceMesh`, `VolumeMesh` from `_core`

### Key Design Patterns

- **Intrusive reference counting**: `Model`, `ModelItem`, `MeshCase`, `Mesh` all use `nb::intrusive_base` so Python and C++ share ownership without leaks.
- **`@nb` annotations in headers**: `src/phnx.h` uses structured doc comments (e.g. `@nb inherit:`, `@nb prop_r:`, `@nb extra:`) that `nanobindgen` parses to generate the binding code. When adding new public methods, follow the existing annotation style.
- **SMS `pPList` ŌåÆ `std::vector`**: The `plist_to_vec<T>()` template handles converting Simmetrix linked list results; it deletes the list.
- **Gmsh export**: `Mesh::write_gmsh()` translates SMS mesh entities (vertexŌåÆedgeŌåÆfaceŌåÆregion) into Gmsh discrete entities, preserving the model topology hierarchy. Physical group names encode metadata as `key=value&key=value` strings (e.g. `tag=5&material=Blanket`).

### Workflow (typical usage)

```python
import phnx
model = phnx.Model.from_parasolid_file("geometry.x_t")   # Load Parasolid assembly
nm_model = model.make_non_manifold_model()                 # Imprint/merge geometry
mesh_case = phnx.MeshCase(nm_model)
mesh_case.set_size(0.1)
surface_mesh = phnx.SurfaceMesh.from_model(nm_model, mesh_case)
volume_mesh = phnx.VolumeMesh.from_surface_mesh(surface_mesh)
volume_mesh.write_gmsh("output.msh")
```

### Simmetrix API Documentation

Doxygen-generated HTML docs for the SMS C API are in `./doc/Simmetrix/` (gitignored). Key sections:

- `doc/Simmetrix/GeomSimParasolid/` ŌĆö Parasolid kernel integration (`ParasolidNM_*`, `GAM_*`)
- `doc/Simmetrix/GeomSim/` ŌĆö Geometric model API (`GM_*`, `GF_*`, `GE_*`, `GV_*`, `GR_*`, `GEN_*`)
- `doc/Simmetrix/MeshSim/` ŌĆö Meshing API (`M_*`, `MS_*`, `SurfaceMesher_*`, `VolumeMesher_*`)
- `doc/Simmetrix/PartitionedMesh/` ŌĆö Partitioned/parallel mesh (`SimPartitionedMesh_*`)

When working with unfamiliar SMS API calls, read the relevant HTML files from `./doc/Simmetrix/`.

### Dependencies

- **Simmetrix SimModSuite** ŌĆö proprietary; headers in `include/`, shared libs in `lib/`. Requires license files (`simmetrix.lic`, `simmodsuite.lic`, `TheaEnergyEval2025`).
- **Gmsh** ŌĆö provided via conda-forge through pixi
- **nanobind** ŌĆö Python binding framework, also via conda-forge
- **spdlog v1.12.0** ŌĆö fetched at configure time via FetchContent
- **nanobindgen** ŌĆö local editable package at `../nanobindgen` for auto-generating bindings
